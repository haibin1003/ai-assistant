# AI 助手管理系统设计文档

## 1. 系统概述

AI 助手是一个独立运行的智能服务，通过 MCP（Model Context Protocol）协议接入目标系统。用户可以通过 Web 界面与 AI 对话，AI 会自动调用已接入系统的工具完成任务。

### 1.1 核心特性

- **独立性**：独立部署的服务，不与任何特定系统耦合
- **多系统接入**：支持同时接入多个目标系统（如 OSRM、其他业务系统）
- **MCP 协议**：通过 MCP 协议与目标系统通信
- **用户上下文**：支持登录和非登录两种模式

---

## 2. 用户角色与权限

| 角色 | 说明 | 权限 |
|------|------|------|
| **游客** | 未登录用户 | 只能使用公开的 MCP 工具（无认证） |
| **用户** | 已登录用户 | 可使用所有已接入系统的 MCP 工具 |
| **管理员** | 管理员用户 | 可配置模型、接入系统、管理用户 |

## 4. 页面设计

### 4.1 页面结构

```
首页 (/)
    ├── 对话页面 (/chat)
    │       └── 聊天界面（用户右，AI左）
    ├── 配置页面 (/settings)
    │       └── Skills 管理
    │       └── 系统接入管理
    └── 管理页面 (/admin) - 仅管理员
            └── 模型配置
            └── 系统注册
```

### 4.2 对话页面

**布局**：
- 左侧：系统列表（已接入的 MCP 系统）
- 右侧：对话区域

**消息布局**：
```
┌────────────────────────────────────┐
│ 👤 用户消息                    →   │  ← 右侧（用户）
│    内容                            │
└────────────────────────────────────┘

┌────────────────────────────────────┐
│ ←   🤖 AI 回答                     │  ← 左侧（AI）
│    内容                            │
│    🔧 工具调用信息                  │
└────────────────────────────────────┘
```

**交互流程**：
1. 用户选择系统（登录状态）
2. 输入消息，发送
3. 后端获取用户上下文
4. 调用 LLM 生成响应
5. 如需工具调用，MCP 执行工具
6. 流式返回响应，前端展示

### 4.3 配置页面

**菜单项**：
1. **Skills 管理**
   - 查看全局 Skills
   - 查看/编辑我的 Skills
   - 创建新 Skill
   - 发布 Skill（设为全局）

2. **系统接入**
   - 查看已接入系统
   - 接入新系统（MCP URL + 认证信息）

## 5. 核心交互流程

### 5.1 用户登录流程

```
用户打开首页 → 选择登录 → 输入 OSRM 账号 → 登录成功
                                              ↓
                                    推送上下文到 AI 助手后端
                                              ↓
                                    创建会话，保存凭证
                                              ↓
                                    可使用全部 MCP 工具
```

**API 调用**：
```http
POST /api/v1/context/push
X-Session-Id: {sessionId}

{
  "sessionId": "xxx",
  "systemId": "osrm",
  "user": {
    "id": 1,
    "username": "admin",
    "roles": ["ADMIN"]
  },
  "credentials": {
    "username": "admin",
    "password": "admin123"
  }
}
```

### 5.2 游客使用流程

```
用户打开首页 → 不登录，直接对话
                      ↓
            选择"不登录使用"
                      ↓
            系统设为游客模式
                      ↓
            只能调用公开工具（无认证 MCP）
```

**权限判断逻辑**：
```
IF user is logged in:
    IF tool is public (portal):
        use portal endpoint (no auth)
    ELSE:
        use api endpoint (with user's auth)
ELSE:
    # 游客
    IF tool is public:
        use portal endpoint
    ELSE:
        return error "请先登录"
```

### 5.3 对话流程

```
用户发送消息 → 前端添加用户消息到列表
                    ↓
           调用 SSE 接口 /api/v1/chat
           X-Session-Id: {sessionId}
                    ↓
           后端获取上下文
           ├─ 已登录：用户信息 + 凭证
           └─ 未登录：游客模式
                    ↓
           构建 System Prompt
           ├─ 注入用户信息（如果已登录）
           └─ 注入可用工具列表
                    ↓
           调用 LLM (DeepSeek/GPT/Claude)
                    ↓
           IF 需要工具调用:
               ToolRouter 执行工具
               ├─ 公开工具 → /portal/mcp
               └─ 认证工具 → /api/mcp + Basic Auth
               返回工具结果给 LLM
               继续调用 LLM 生成回复
                    ↓
           流式返回 SSE 响应
           ├─ content: 文本内容
           ├─ tool_use: 工具调用
           ├─ tool_result: 工具结果
           └─ done: 完成
                    ↓
           前端解析并展示
```

### 5.4 Skill 管理流程

```
用户进入配置页面 → 选择"Skills"
                    ↓
           显示两个 Tab：
           ├─ 全局 Skills（所有人可用）
           └─ 我的 Skills（自己创建的）
                    ↓
           创建新 Skill:
           ├─ 填写基本信息
           │  ├─ skillId (唯一标识)
           │  ├─ name (显示名称)
           │  ├─ description (描述)
           │  └─ promptTemplate (提示词)
           ├─ 配置触发词（可选）
           └─ 配置所需工具（可选）
                    ↓
           保存 → 自己的 Skills 列表
                    ↓
           如需发布 → 点击"发布"
           → 设为全局，其他人也可用
```

## 6. 权限与安全

### 6.1 游客权限限制

| 功能 | 游客 | 登录用户 |
|------|------|----------|
| 对话 | ✅ | ✅ |
| 调用公开 MCP 工具 | ✅ | ✅ |
| 调用认证 MCP 工具 | ❌ | ✅ |
| 创建 Skill | ❌ | ✅ |
| 管理已接入系统 | ❌ | ✅ |
| 接入新系统 | ❌ | ❌ (仅管理员) |

### 6.2 会话管理

- Session 存储在 MySQL
- 默认 24 小时过期
- 支持会话续期
- 游客会话不保存凭证

## 8. API 设计

### 8.1 对话 API

```http
# 流式对话
POST /api/v1/chat
Content-Type: application/json
X-Session-Id: {sessionId}

{ "content": "你好" }

# 响应 (SSE)
data: {"event":"content","data":{"content":"你好！"}}
data: {"event":"tool_use","data":{"tool":"osrm_search","args":{...}}}
data: {"event":"tool_result","data":{"success":true,"data":"..."}}
data: {"event":"done","data":{}}
```

### 8.2 上下文推送 API

```http
# 登录时推送用户上下文
POST /api/v1/context/push

{
  "sessionId": "xxx",
  "systemId": "osrm",
  "user": { "id": 1, "username": "admin", "roles": ["ADMIN"] },
  "credentials": { "username": "admin", "password": "xxx" }
}
```

### 8.3 Skills API

```http
GET  /api/v1/skills          # 获取可用技能
GET  /api/v1/skills/global   # 获取全局技能
GET  /api/v1/skills/my       # 获取我的技能
POST /api/v1/skills          # 创建技能
PUT  /api/v1/skills/{id}     # 更新技能
DELETE /api/v1/skills/{id}   # 删除技能
```

### 8.4 系统管理 API

```http
GET    /api/v1/systems              # 获取已接入系统
POST   /api/v1/systems               # 接入新系统 (管理员)
DELETE /api/v1/systems/{id}          # 删除系统
GET    /api/v1/systems/{id}/tools     # 获取系统工具列表
```

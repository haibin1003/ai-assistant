---

## 1. 项目概述

### 1.1 项目背景

OSRM（开源软件仓库管理）系统已具备完整的业务功能和 MCP 服务接口。为了提升用户体验，需要一个独立的 AI 助手服务，通过自然语言对话帮助用户完成软件管理、订购申请等操作。

### 1.2 项目目标

构建一个**独立的、可插拔的 AI 助手服务**：
- 通过 MCP 协议接入任何目标系统
- 用户通过自然语言对话完成业务操作
- 支持多系统并发接入
- 前端可嵌入目标系统或独立部署

### 1.3 核心价值

| 价值点 | 说明 |
|--------|------|
| 独立性 | 作为独立服务部署，不与任何特定系统耦合 |
| 可插拔 | 任何提供 MCP 服务的系统都可直接接入 |
| 智能化 | AI 理解用户意图，自动调用系统工具完成任务 |
| 上下文感知 | 自动获取用户角色权限，提供个性化服务 |

### 3.2 功能详细说明

#### 3.2.1 系统注册管理

**需求描述**：管理可接入的目标系统

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 注册系统 | 添加新的目标系统配置（系统ID、名称、MCP网关地址、认证方式、工具前缀） | P0 |
| 更新系统 | 修改已注册系统的配置 | P0 |
| 注销系统 | 移除不再使用的系统 | P1 |
| 工具发现 | 调用 MCP tools/list 获取系统提供的工具列表 | P0 |
| 工具缓存 | 缓存工具列表，定期刷新（默认5分钟） | P1 |

**系统注册信息**：

```json
{
  "systemId": "osrm",
  "systemName": "开源软件仓库管理",
  "iconUrl": "https://osrm.example.com/icon.png",
  "mcpGatewayUrl": "http://osrm-mcp:3000/api/mcp",
  "authType": "bearer",
  "toolPrefix": "osrm_",
  "description": "OSRM 系统描述"
}
```

#### 3.2.3 对话引擎

**需求描述**：处理用户与 AI 的对话交互

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 流式对话 | 支持 SSE/WebSocket 流式响应 | P0 |
| 多模型支持 | 支持 DeepSeek / OpenAI / Claude 等大模型 | P0 |
| 工具调用 | AI 自动判断并调用 MCP 工具 | P0 |
| 上下文注入 | 自动注入用户上下文到 System Prompt | P0 |
| 历史消息 | 保存和管理对话历史 | P1 |
| 对话管理 | 创建、查询、删除对话 | P1 |

**对话请求**：

```json
POST /api/v1/chat
Headers: X-Session-Id: {sessionId}
Body: {
  "content": "帮我上传一个 Nginx 软件"
}
```

**流式响应 (SSE)**：

```
event: content
data: {"content": "好的，我来帮您上传 Nginx 软件。"}

event: tool_use
data: {"tool": "osrm_search_software", "args": {"keyword": "nginx"}}

event: tool_result
data: {"success": true, "data": [...]}

event: done
data: {}
```

#### 3.2.4 MCP 工具路由

**需求描述**：将 AI 的工具调用路由到正确的目标系统

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 工具名解析 | 解析工具名前缀，确定目标系统 | P0 |
| 认证注入 | 注入用户的访问令牌到 MCP 请求 | P0 |
| 工具调用 | 调用目标系统的 MCP tools/call | P0 |
| 结果处理 | 处理工具调用结果，返回给 AI | P0 |

**工具名格式**：`{toolPrefix}{toolName}`

示例：
- `osrm_search_software` → 系统: OSRM, 工具: search_software
- `systemb_create_order` → 系统: SystemB, 工具: create_order

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 对话首字响应时间 | < 2 秒 |
| 工具调用延迟 | < 5 秒 |
| 并发用户数 | > 100 |
| 会话缓存命中率 | > 80% |

### 4.2 安全需求

| 需求 | 说明 |
|------|------|
| API Key 加密存储 | 用户配置的 API Key 需加密存储 |
| 访问令牌安全 | 目标系统的访问令牌加密存储 |
| 会话隔离 | 不同用户的会话数据完全隔离 |
| 日志脱敏 | 日志中不记录敏感信息 |

### 4.3 可用性需求

| 需求 | 说明 |
|------|------|
| 服务可用性 | 99.9% |
| 故障恢复 | 自动重连 MCP 网关 |
| 健康检查 | 提供健康检查接口 |

## 6. 集成需求

### 6.1 与目标系统集成

**集成方式**：目标系统通过 HTTP 调用 AI 助手 API

**集成步骤**：

1. **注册系统**：目标系统管理员调用 `POST /api/v1/systems` 注册系统信息
2. **嵌入对话界面**：目标系统前端嵌入 AI 助手组件
3. **推送上下文**：用户登录后，目标系统调用 `POST /api/v1/context/push`
4. **用户对话**：用户通过嵌入的界面与 AI 对话
5. **清理上下文**：用户登出时，目标系统调用 `DELETE /api/v1/context/{sessionId}`

### 6.2 前端嵌入方式

**方式一：Vue 组件**

```vue
<AIChatWidget
  apiBaseUrl="http://ai-assistant:8081"
  systemId="osrm"
  :userContext="userContext"
/>
```

**方式二：iframe**

```html
<iframe src="http://ai-assistant:8081/embed?system=osrm" />
```
